# KAIROS MCP logo — requirements brief

Use this brief when commissioning logo mockups from agencies or designers. It keeps submissions comparable and on-brand.

---

## 1. Product and brand

**Product name:** **KAIROS MCP** (not “Kairos” alone).

**What it is:** An MCP (Model Context Protocol) server for persistent memory and deterministic protocol-chain execution. AI agents run workflows via MCP or CLI; humans use the product to browse, view, create, and manage protocols.

**Brand traits:** Professional, precise, trustworthy. Technical but not cold. Avoid playful or consumer-heavy styling.

---

## 2. Naming and differentiation

- **Always use “KAIROS MCP”** in the logo or lockup. “KAIROS” alone is used by many others (time, photography, other tech). We need a distinct, ownable mark.
- Acceptable lockups: **KAIROS MCP**, **KAIROS · MCP**, **KAIROS** with “MCP” as a tagline or subscript. Do not submit “Kairos” without “MCP” as the primary product mark.
- Optional: symbol or icon that can stand alone (e.g. favicon, app icon) and that we can pair with the wordmark.

---

## 3. Deliverables to quote for

Please provide mockups and final files for:

| Item | Description |
|------|-------------|
| **Primary wordmark** | “KAIROS MCP” (or approved lockup). Main use: app header, docs, marketing. |
| **Icon / symbol** | Standalone mark (no text). Use: favicon, app icon, sidebar, social. Must read at 32×32 px. |
| **Horizontal lockup** | Symbol + wordmark in one horizontal unit for headers. |
| **Clear space and minimum size** | Specify minimum size and clear space rules. |

Formats we need for production:

- **Vector:** SVG (preferred) or EPS/AI for logo and icon.
- **Raster (from vector):** PNG with transparency at 1x and 2x for at least 32, 64, 128, 256 px (icon) and a header-sized width for wordmark (e.g. 200 px height).
- **Favicon:** ICO or PNG 32×32, 48×48 (optional).

---

## 4. Colour

- **Primary:** Teal `#0d9488` (use for primary logo or accent).
- **Dark:** `#0f172a` (headings, dark variant of logo).
- **Supporting:** Logo must work on white `#ffffff` and on light grey `#f8fafc`. Optionally provide a single-colour (e.g. dark) version for light backgrounds and a light version for dark backgrounds.

Contrast must meet **WCAG 2.2 AA** where the logo appears on UI (e.g. text or logo next to background ≥ 4.5:1 for normal text, 3:1 for large areas).

---

## 5. Usage context

Logo will be used in:

- Web app (header, sign-in redirect, favicon).
- Documentation and README.
- Optional: GitHub, social, installers or packaging.

Avoid:

- Clock or “time” imagery (conflicts with other Kairos brands).
- Overly generic tech (generic circuit boards, globes). A subtle “chain” or “protocol” cue is acceptable if it stays minimal.

---

## 6. What to submit per concept

For each concept, please provide:

1. **Primary wordmark** (KAIROS MCP) on white and on light grey.
2. **Icon only** at 32×32 and 128×128 (or 256×256).
3. **One-sentence concept** describing the idea (e.g. “Linked nodes suggest protocol chains”).
4. **Clear space / minimum size** note.

Final chosen concept: full vector + raster set as in §3.
