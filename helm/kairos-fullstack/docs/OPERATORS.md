# Step-by-step operator installation

Install these operators **before** installing the KAIROS full stack Helm chart when using chart-created clusters. See [Keycloak operator installation](https://www.keycloak.org/operator/installation) for official references.

---

## 1. Redis operator (Spotahome)

Uses the **RedisFailover** CRD. The latest Helm chart (3.3.x) has a CRD template bug; use **chart version 3.1.6**.

```bash
helm repo add redis-operator https://spotahome.github.io/redis-operator
helm repo update
kubectl create namespace redis-operator
helm install redis-operator redis-operator/redis-operator -n redis-operator --version 3.1.6
```

Verify: `kubectl get pods -n redis-operator` and `kubectl get crd redisfailovers.databases.spotahome.com`.

---

## 2. Keycloak operator

Use the **new** operator (CRD `keycloaks.k8s.keycloak.org`). CRDs are separate from the operator deployment.

```bash
kubectl create namespace keycloak
export KEYCLOAK_OPERATOR_VERSION=26.5.4

# CRDs (cluster-scoped)
kubectl apply -f "https://raw.githubusercontent.com/keycloak/keycloak-k8s-resources/${KEYCLOAK_OPERATOR_VERSION}/kubernetes/keycloaks.k8s.keycloak.org-v1.yml"
kubectl apply -f "https://raw.githubusercontent.com/keycloak/keycloak-k8s-resources/${KEYCLOAK_OPERATOR_VERSION}/kubernetes/keycloakrealmimports.k8s.keycloak.org-v1.yml"

# Operator in keycloak namespace
kubectl -n keycloak apply -f "https://raw.githubusercontent.com/keycloak/keycloak-k8s-resources/${KEYCLOAK_OPERATOR_VERSION}/kubernetes/kubernetes.yml"
```

Verify: `kubectl get pods -n keycloak` and `kubectl get crd keycloaks.k8s.keycloak.org`.

**Note:** The chart's Keycloak CR template uses `keycloak.org/v1alpha1` (legacy). If you install via the steps above, use `keycloakInstance.useOwnCluster: true` and create your Keycloak CR manually with the new CR spec, or install the legacy operator via OLM.

---

## 3. Percona PostgreSQL operator

```bash
helm repo add percona https://percona.github.io/percona-helm-charts/
helm repo update
kubectl create namespace percona-pg
helm install pg-operator percona/pg-operator -n percona-pg
```

Verify: `kubectl get pods -n percona-pg` and `kubectl get crd perconapgclusters.pgv2.percona.com`.

---

## 4. Install KAIROS full stack

```bash
cd helm/kairos-fullstack
helm dependency update
helm install kairos . -n kairos --create-namespace -f my-values.yaml
```

Set `app.redisUrl`, `app.qdrantUrl`, `app.keycloakInternalUrl` (and optionally enable `redisCluster.enabled` / `keycloakInstance.enabled` / `postgresCluster.enabled` so the chart creates the CRs). Credentials are autogenerated and stored only in the Kubernetes Secret unless you set `credentials.existingSecret`.
