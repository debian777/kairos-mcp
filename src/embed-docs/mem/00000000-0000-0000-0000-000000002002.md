# Get help refining your search

**You are an AI agent.** You ran **`activate`** and got no solid match (or only weak/ambiguous ones). This adapter helps you turn the user's vague request into a better query so the next **`activate`** can find the right adapter.

## Activation Patterns

**Run this adapter when:**

- **activate** returned no match, or only choices with low **activation_score** (e.g. all &lt; 0.5).
- The response included a **refine** choice directing you to improve the query.

**Trigger pattern:** After **activate** when result is no strong match / refine suggested.

**Must Never:**

- Run when a strong match (score ≥ 0.5) already exists — use that choice and **forward** instead.
- Run when the user has not yet asked for anything (no prior activation context).

**Must Always:**

- Run at most once per user request — if refining twice still yields no match, stop and offer **train** or ask the user to clarify.

**Good trigger examples:**

- **activate** returned only "refine" / weak matches → run this adapter
- "No protocol matched 'do the thing'" → run this adapter

**Bad trigger examples:**

- **activate** returned a strong match → use that match, not this adapter
- User asked "what adapters exist?" → use **activate**/**spaces**, not this adapter

## Step 1: Extract what the user actually wants

From the user's original message, identify goal, context, and gaps.

Write your analysis as your solution (goal + context + gaps, minimum 30 characters).

```json
{
  "contract": {
    "type": "comment",
    "comment": { "min_length": 30 },
    "required": true
  }
}
```

## Step 2: Build and run a refined activate query

Using your Step 1 analysis, construct a single query (3–8 specific words). Do not reuse the vague phrase that already failed.

Call **`activate`** with it.

- **Strong match (score >= 0.5):** Pick that choice and **forward** with its adapter URI.
- **Weak matches only:** Ask the user to clarify, or pick the create choice to **train** a new adapter.

Do not loop more than once — if two activations fail, the adapter likely does not exist yet.

```json
{
  "contract": {
    "type": "mcp",
    "mcp": { "tool_name": "activate" },
    "required": true
  }
}
```
