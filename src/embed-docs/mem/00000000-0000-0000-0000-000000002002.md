# Get help refining your search

You ran `kairos_search` and received no solid match (or only weak or
ambiguous results). This protocol turns the user's vague request into a
more specific query so the next `kairos_search` can find the right protocol.

## Step 1: Extract what the user actually wants

From the user's original message, identify:

- **Goal:** What outcome do they want? (e.g. "run tests", "write docs",
  "create a Jira ticket")
- **Context:** Any project, tool, domain, or proper noun they mentioned.
- **Gaps:** What is missing or ambiguous? (e.g. "which repo?", "which
  doc type?", "which project key?")

Expand slash-commands or shorthand (e.g. `/ai-docs` → "write ai
instructions zero-drift template").

Write your analysis as your solution (goal + context + gaps, minimum 30
characters).

```json
{
  "challenge": {
    "type": "comment",
    "comment": { "min_length": 30 },
    "required": true
  }
}
```

## Step 2: Build and run a refined kairos_search query

Using your Step 1 analysis, construct a single search query:

- Use 3–8 specific words (nouns and verbs, no filler). Examples:
  "write ai instructions zero-drift template", "jira create story BIB",
  "npm test lint deploy".
- Include domain or tool names when known (e.g. "jira", "gitlab",
  "coding standards").
- Do not reuse the exact vague phrase from the first failed search.

Write your refined query as your solution, then call `kairos_search`
with it.

- **Strong match (score ≥ 0.7):** Pick that choice; call `kairos_begin`
  with its URI.
- **Weak matches only (all scores < 0.7):** Ask the user to clarify, or
  pick the create choice to build a new protocol.

Run this refining protocol at most once. When two searches fail, the
protocol does not exist yet — pick the create choice.

```json
{
  "challenge": {
    "type": "mcp",
    "mcp": { "tool_name": "kairos_search" },
    "required": true
  }
}
```
