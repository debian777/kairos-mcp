Two complementary layers:
- `helm/.dev/` — Bash entry point (`helm-deploy.sh`) plus staged `values-*.yaml` overlays (default → http → tls → tls-redis → full) that progressively enable gateway, Valkey, Keycloak operator, Percona PG operator, and Ollama. The script auto-inferes the current release's profile from Helm values, installs Traefik GatewayClass + HelmChartConfig for Kubernetes Gateway API, conditionally boots Keycloak and Percona operators (or falls back to ArgoCD kustomizations if present), creates the `kairos-mcp-embedding` secret from `OPENAI_API_KEY`, then runs `helm upgrade --install` with the selected overlay.
- `helm/values.dev.yaml` / `helm/values.prod.yaml` — environment-level value overrides (dev pins `latest` image tag, enables ngrok gateway route and Ollama; prod adds HPA, resource requests/limits, and references an existing embedding secret). Both are consumed by the same chart via `-f` on the CLI.
The dependency direction is one-way: scripts depend on the chart and its values schema; the chart itself has no knowledge of these dev helpers.
