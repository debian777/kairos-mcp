Pure documentation-based skills consumed by the vercel-labs/skills CLI and agent runtimes (Claude Code, Cursor). Each skill is a directory with a required `SKILL.md` containing YAML frontmatter (`name`, `description`, optional `metadata`, `allowed-tools`) and a markdown body — no executable code lives here. Two discovery locations are used: `skills/kairos/` for the everyday routing skill and `skills/.system/` for helper skills (`kairos-bug-report`, `kairos-install`), per the skills CLI spec. `README.md` documents installation (`npx skills add debian777/kairos-mcp [--skill ...]`) and usage tiers; `SKILLS.md` is the authoring reference aligned to agentskills.io/specification and points at the `skills-ref` validator used in CI via `npm run lint:skills`. Dependency direction is one-way: these files instruct an external MCP server's tools (`activate`, `forward`, `reward`, `train`, etc.) but contain no runtime logic themselves.
