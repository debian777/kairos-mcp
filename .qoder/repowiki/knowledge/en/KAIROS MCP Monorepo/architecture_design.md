Root `package.json` is the single source of truth: it declares every child as a workspace dependency, wires `bin.kairos` → `cli/dist/index.js`, `main` → `shared_runtime/dist/index.js`, and lists the root `build` script (`prebuild → embedded-docs build → skills version sync → ui:build → tsc`) that emits a flat `dist/` tree published as `@debian777/kairos-mcp`. Runtime wiring between children is environment-driven — `.env.*` profiles (mini/fullstack/infra-ui) plus `scripts/deploy-run-env.sh` select transports and compose stacks, which the bootstrap in `shared_runtime` consumes to start either stdio or the Express `http_api` server. The same env surface is consumed uniformly at build time by `embedded_docs` and `mcp_apps` so generated resources stay in lockstep with configuration. Dockerfiles and `compose.yaml` pin the app image tag to the npm version via `compose:sync-app-tag`, keeping containerised deployments aligned with the published package.
