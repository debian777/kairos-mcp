Flat Markdown tree organized by audience/topic rather than code layout: `architecture/` holds system design and per-MCP-tool companion pages (`workflow-*.md`) that mirror the canonical tool surface in `src/embed-docs/tools/`; `install/` groups deployment guides (Docker Compose profiles, Helm, prerequisites); `security/` contains threat model, incident runbook, and code-security setup; `examples/` documents adapter authoring with runnable example adapters; `business/` records operational case studies; `specs/` keeps parity contracts (e.g. artifact-export-parity-spec). Top-level files (`README.md`, `CLI.md`, `kairos-bundles.md`, test-snapshot docs) act as entry points and cross-reference sub-sections via relative links. There is no build step — content is consumed directly by readers and referenced from other repo docs.
