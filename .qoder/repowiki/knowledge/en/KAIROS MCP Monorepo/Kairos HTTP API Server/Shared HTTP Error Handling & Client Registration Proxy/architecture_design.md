Three independent Express-level utilities with no internal dependencies on each other:
- `http-error-handlers.ts` exports `setupErrorHandlers(app)` which registers an Express `ErrorRequestHandler` (global 500/413 mapper via `structuredLogger`), a `/mcp` GET→405 guard, and a trailing catch-all 404.
- `http-route-errors.ts` exposes `sendToolRouteError(res, error, fallbackErrorCode)` plus a local `resolveHttpStatus` helper; it translates domain `KairosError` instances into `{error, message, ...details}` JSON payloads so route handlers can return typed errors without touching Express directly.
- `http-client-registration-proxy.ts` exports `setupClientRegistrationProxy(app)` which mounts the OpenID Connect Dynamic Client Registration prefix `/.well-known/clients-registrations/openid-connect` and forwards POST/GET/PUT/DELETE to `${KEYCLOAK_INTERNAL_URL}/realms/${KEYCLOAK_REALM}/clients-registrations/openid-connect`, rewriting `registration_client_uri` and `Location` headers to replace the internal Keycloak host with `AUTH_CALLBACK_BASE_URL`. It uses Node's native `fetch` with a 10s `AbortController` timeout and returns 502 on upstream failure.

Dependency direction: all three depend outward on `../config.js` (Keycloak URLs) and `../utils/structured-logger.js`; only `http-route-errors.ts` depends on the shared `KairosError` type from `../types/index.js`. Each file is a pure module — callers explicitly invoke the `setup*` function against their Express app instance.
