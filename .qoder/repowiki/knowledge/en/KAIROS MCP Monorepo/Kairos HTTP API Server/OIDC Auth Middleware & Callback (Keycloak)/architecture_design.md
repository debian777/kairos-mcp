Three cooperating files form the auth boundary around protected Express routes:
- `http-auth-middleware.ts` exports `authMiddleware`, an Express middleware that short-circuits unauthenticated requests to `/api`, `/mcp`, and `/ui`. It accepts two auth paths: (1) a signed HMAC-SHA256 session cookie (`kairos_session`) parsed via `getSessionPayload`, and (2) an Authorization: Bearer JWT validated by `validateBearerToken`. On success it attaches `req.auth` (an `AuthPayload`) and runs the handler inside `runWithSpaceContext`; on failure it returns 401 with `WWW-Authenticate` (and a JSON-RPC 2.0 error envelope for MCP), or 302 redirects browser GETs to the OIDC login.
- `bearer-validate.ts` implements stateless JWT verification using `jose`: decodes `iss`, checks it against `AUTH_TRUSTED_ISSUERS`, fetches the issuer's JWKS from `/protocol/openid-connect/certs` (cached per-issuer in a process-level `Map`), verifies signature/exp/audience, then enriches groups either from the access token, a nested `id_token`, or a fallback call to the OIDC userinfo endpoint — all filtered through `applyOidcGroupsAllowlist`.
- `http-auth-callback.ts` registers the `/auth/callback` route (plus `/auth`, `/auth/logout`, `/auth/logged-out`, `/auth/continue-signin`). It exchanges the authorization code for tokens via Keycloak's token endpoint, merges id_token/access_token payloads, computes a session TTL derived from token expiry, signs the payload into the same HMAC cookie format used by the middleware, and sets it as an httpOnly SameSite=Lax cookie.

Dependency direction is one-way: callback → middleware (for `peekOidcIdTokenHintFromSession` + `SESSION_COOKIE_NAME`); middleware → bearer-validate; both rely on shared helpers in `oidc-profile-claims.ts` and config in `../config.js`. The module never touches a database — identity is carried purely in the signed cookie and the inbound Bearer token.
