Three sibling Express helpers registered at app bootstrap:
- `setupHealthRoutes(app, memoryStore)` mounts `/health` (liveness/readiness) and `/api` (service catalog). Readiness probes Qdrant via `MemoryQdrantStore.checkHealth`, Redis via `keyValueStore.isConnected()`, and the embedding provider through a dual-path call (`teiHealthCheck` or `healthCheck`) wrapped in a 2s `Promise.race` timeout so upstream failures never block the response. Criticality is computed as `qdrant && (!redis || redisHealthy) && (!embedding || teiHealthy)` and mapped to 200/503.
- `httpMetricsMiddleware` is an Express middleware that instruments every request with four Prometheus collectors from `../services/metrics/http-metrics.js`: `httpRequests` (inc on finish), `httpRequestDuration` (startTimer + label set on end), `httpRequestSize` (from `Content-Length` header), `httpResponseSize` (wraps `res.send` to capture body length), and `httpActiveConnections` (inc at entry, dec on finish). All labels include `method`, `route`, `status`, and `tenant_id` (from `getTenantId`).
- `setupWellKnown(app)` registers RFC 9728 Protected Resource Metadata at `/.well-known/oauth-protected-resource` and `/.well-known/oauth-protected-resource/mcp`, plus an RFC 8414 Authorization Server Metadata proxy at `/.well-known/oauth-authorization-server` and `/.well-known/openid-configuration`. The PRM is built by `buildProtectedResourceMetadata()` using `AUTH_CALLBACK_BASE_URL`, `KEYCLOAK_*`, and `OIDC_SCOPES_SUPPORTED`; the ASM metadata is fetched once per process from Keycloak's `/.well-known/openid-configuration`, cached for 300s, and rewritten to expose the external URL while preserving `registration_endpoint` for DCR. A separate `setupClientRegistrationProxy(app)` helper is imported to handle client registration. Both handlers emit CORS headers when `Origin` is present.
