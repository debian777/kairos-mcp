The module is an Express application assembled by `http-server.ts` / `http-server-startup.ts`, which registers middleware (OIDC session/Bearer validation from `auth_oidc`, global error handling and structured tool-route errors from `shared_http_utils`, Prometheus metrics, CORS for `/mcp`) and then mounts route groups: REST endpoints under `/api/*` (`rest_api_routes`), MCP JSON-RPC at POST `/mcp` (`mcp_jsonrpc`), SPA assets under `/ui` (`ui_static_auth`), discovery/health/well-known endpoints (`health_metrics_wellknown`), and time-limited export downloads (`export_download`). Cross-cutting concerns — OIDC profile claims, scope parsing, WWW-Authenticate header construction, client-registration proxying, and audit event emission — are shared across all route groups via the sibling utility modules rather than being duplicated per handler.
