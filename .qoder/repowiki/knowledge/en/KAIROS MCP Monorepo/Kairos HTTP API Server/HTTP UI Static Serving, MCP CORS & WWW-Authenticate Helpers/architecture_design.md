Three independent Express setup helpers with no cross-imports, each mounted by the HTTP server entry point:
- `http-ui-static.ts` — resolves the SPA root via `KAIROS_UI_DIR` (falling back to `dist/ui` next to `src/http` or two levels up when running from TypeScript sources), detects whether the built bundle includes browse-by-letter code by scanning `assets/KairosPage-*.js`, then mounts `express.static('/ui')` plus an SPA fallback route that rewrites unmatched GET requests to `index.html`. Rate-limiting is applied through `createRateLimiter('ui-static', ...)` using shared `HTTP_RATE_LIMIT_*` config.
- `http-www-authenticate.ts` — pure value builders (`buildWwwAuthenticate`, `setWwwAuthenticate`) that compose a `Bearer realm="mcp"` header including `resource_metadata`, optional `authorization_uri` (from `KEYCLOAK_URL`/`KEYCLOAK_REALM`), and `scope`; uses `error=invalid_token` so MCP clients clear stale tokens after Keycloak session cleanup. No Express state touched.
- `http-mcp-cors.ts` — lightweight `/mcp` preflight + request interceptor that mirrors the incoming `Origin` into `Access-Control-Allow-Origin` (with `Vary: Origin`) and explicitly exposes the `WWW-Authenticate` header so browsers can surface it to MCP clients; OPTIONS returns 204 with allowed methods/headers.
Dependency direction is one-way: these files import shared config (`../config.js`) and logger/rate-limiter utilities, but nothing in this module imports higher-level application logic.
