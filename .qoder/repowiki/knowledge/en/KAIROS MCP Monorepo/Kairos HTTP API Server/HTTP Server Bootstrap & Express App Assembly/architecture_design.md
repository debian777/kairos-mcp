Three-file bootstrap split by concern:
- `http-server-config.ts` owns cross-cutting middleware: structured access logging (`httpLogger`), Prometheus metrics (`httpMetricsMiddleware`), Helmet CSP (minimal `'self'` policy), per-prefix rate limiting via a shared `createRateLimiter` factory for `/auth`, `/api`, `/mcp`, and `express.json` body parsing with a configured size limit.
- `http-server.ts` is the composition root: it creates the Express app, registers middleware in a strict order (well-known before auth so RFC 9728 discovery is unauthenticated, then CORS, then auth, then routes), injects `MemoryQdrantStore` and `qdrantService` into route setup functions, and delegates lifecycle to the startup module. It also exposes `startHttpTransport` as the public entry point used by the transport layer.
- `http-server-startup.ts` isolates the OS-level `app.listen('0.0.0.0', port)` call plus an `EADDRINUSE` / generic error handler that logs via `structuredLogger` and calls `process.exit(1)`, returning the raw `http.Server` instance.
Dependency direction is one-way: config → startup ← server; route modules are imported only by the server file, keeping this module free of business logic.
