Two sibling Node/TypeScript build scripts under `scripts/` walk the source tree rooted at `src/embed-docs/`:
- `build-embed-docs.ts` is the entry point. It reads every top-level subdirectory of `src/embed-docs/`, flattens `.md` files from `prompts/`, `tools/`, `templates/` into flat key→content maps, preserves nesting inside `resources/`, collects YAML frontmatter `slug:` entries from `mem/` and root `.md` files via `collectMetaBySlug`, then writes `src/resources/embedded-mcp-resources.ts` — a self-contained TS module exporting `mcpResources` plus typed getters (`getPrompt`, `getResource`, `getTemplate`, `getToolDoc`, `getMetaDoc`, `listResourceKeys`).
- `build-embed-docs-slug-meta.ts` is a pure helper: `extractSlugFromMarkdown` parses the first YAML frontmatter block for a `slug:` key; `collectMetaBySlug` walks a directory, skips `readme.md`, deduplicates by slug (last-write-wins with a warning), and populates the shared meta map.
- `mem/` is treated specially: its markdown is NOT embedded into the generated TS file but is instead copied verbatim into `dist/embed-docs/mem/` so the running server can read it from disk. Any unknown top-level directory besides `prompts|tools|resources|templates|mem` is auto-discovered and added as another flat category.
- Dependency direction is one-way: both scripts depend only on Node `fs`/`path`; the generated artifact is consumed by the MCP server at runtime and has no dependency back on these scripts.
