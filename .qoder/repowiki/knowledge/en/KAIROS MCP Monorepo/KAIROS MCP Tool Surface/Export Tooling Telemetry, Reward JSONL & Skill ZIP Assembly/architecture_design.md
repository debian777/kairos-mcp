Three sibling helpers consumed by the export orchestration layer:
- `export-telemetry.ts` provides an `AsyncLocalStorage`-backed per-call context (`runExportTelemetryContext`) so concurrent exports keep isolated `decodedBytes` accounting; `finalizeExportObservation` records metrics via `recordExportMetrics` and emits structured logs (slow threshold at 5000 ms) for both generic and `skill_zip` formats.
- `export-reward-jsonl.ts` converts `TrainingPair[]` from the execution trace store into three downstream shapes — a flat `RewardJsonlItem` per pair, an SFT chat-style `{messages:[user,assistant]}` format filtered by `isRewardEligibleForSft`, and a preference format grouped by canonicalized `kairos://layer/<id>:index` picking the highest-scoring success vs. failure pair per group; `canonicalLayerUri` normalizes URIs before grouping.
- `export-skill-items.ts` is the single entry point for building merged skill items for `skill_tree`/`skill_zip`/HTTP binary ZIP: it resolves each adapter URI, assembles the SKILL.md + artifacts + diagnostics, dedupes slugs via `dedupeSlug`/`rehomeItemFiles`, appends artifact files and SHA-256 sums, and returns `{items, primaryUri}`.
Dependency direction is inward toward `services/*`, `types/memory.js`, and the sibling `skill-export/` and `export-selection.js` modules; this leaf exposes only pure transformation / instrumentation functions with no HTTP or CLI wiring.
