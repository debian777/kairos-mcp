Three sibling files each export a pair of `z.infer`-derived TypeScript types (`*Input`, `*Output`) plus an optional `*Solution` type, one per tool. Schemas are built by composing smaller reusable pieces — shared sub-schemas like `adapterUriSchema`, `layerUriSchema`, `forwardFirstCallSchema`, `linkedArtifactSchema`, `tensorContractSchema`, and `solutionTemplateSchema` — then extended via `.extend()` or wrapped in discriminated unions (e.g. `activateChoiceCommonSchema` with `role: 'match'|'refine'|'create'`; `forwardNextCallSchema` with `kind: 'forward'|'reward'`). Cross-file reuse is limited to URI regexes imported from `./kairos-uri.js` and search-limit constants from `../config.js`. Validation goes beyond basic shape checks: `superRefine` enforces cross-field rules (e.g. `solution` forbidden on start calls but required on continuation), `z.preprocess` normalizes legacy v1 solutions into the canonical v2 envelope (`outcome` + `evidence`), and `.strict()` / `.passthrough()` control unknown-key handling per schema. The module exposes no runtime logic — it is consumed by the tool handlers elsewhere in `src/tools/`.
