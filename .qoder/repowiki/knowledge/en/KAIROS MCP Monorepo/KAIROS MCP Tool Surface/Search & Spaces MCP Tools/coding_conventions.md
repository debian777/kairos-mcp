- Each tool exports a plain `executeXxx` function that takes injected dependencies (`MemoryQdrantStore`, optional `QdrantService`) and returns typed data, while a separate `registerXxxTool` function wires it into the MCP server with schema validation and metrics.
- User-facing queries are normalized before cache-keying or storage lookup by stripping built-in protocol URIs/slugs (`queryForSearch`, slug normalization via `normalizeAuthorSlug`/`slugifyFromTitle`) so identical semantics produce identical keys.
- Scores are normalized to `[0,1]` through `normalizePublicSearchScore` before being exposed to consumers, and deduplication uses a Map keyed by `adapter.id || memory_uuid` with head-layer and personal-space tie-breaking rules.
- Optional enrichment calls (chain-root resolution, footer protocol version fetch) are wrapped in try/catch that logs via `structuredLogger.warn` and falls back to defaults rather than throwing, keeping search robust against downstream failures.
- MCP tool handlers wrap body execution in Prometheus instrumentation (`mcpToolCalls.inc`, `mcpToolDuration.startTimer`, `mcpToolErrors.inc`, size observers) and return a uniform `{content, structuredContent}` shape with `isError: true` on failure.
