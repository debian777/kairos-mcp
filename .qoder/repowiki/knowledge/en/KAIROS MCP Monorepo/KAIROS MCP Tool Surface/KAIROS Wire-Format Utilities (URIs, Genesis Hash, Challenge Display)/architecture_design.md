Three independent leaf utilities with no internal sub-packages:
- `kairos-uri.ts` defines the canonical `kairos://adapter/{uuid|slug}`, `kairos://artifact/{uuid|slug}`, and `kairos://layer/{uuid}[?execution_id=…]` schemes via exported regex constants (`ADAPTER_URI_INPUT_REGEX`, `ARTIFACT_URI_INPUT_REGEX`, `LAYER_URI_INPUT_REGEX`) plus a discriminated-union `ParsedKairosUri` type. It provides symmetric `build*Uri` constructors and a `parseKairosUri`/`parseKairosUriOrThrow` pair that delegates to `normalizeAuthorSlug` from `../utils/protocol-slug.js`; an `assertWireAdapterUri` guard enforces the adapter scheme at tool-input boundaries. A transitional legacy layer-row prefix (`kairos://me/m/…`) is handled as a compatibility path.
- `kairos-genesis-proof-hash.ts` exports a single constant `GENESIS_HASH`, computed once at module load by SHA-256-hashing the literal string `'genesis'` via Node's `crypto` — this value seeds step 1 of the proof chain.
- `kairos-challenge-display.ts` consumes `ProofOfWorkDefinition` from `../types/memory.js` and the `GENESIS_HASH` above to build a flat `Record<string, unknown>` challenge shape suitable for MCP output schemas. It reuses shell-invocation helpers from `./shell-challenge-invocation.js` (`buildShellChallengeArgv`, `formatShellInvocationForDisplay`, `pickShellChallengeFields`) and coerces unknown stored `proof.type` values to `comment` while logging via `structuredLogger.warn`.

Dependency direction is strictly inward: this module depends on `../types/*`, `../utils/*`, and sibling `./shell-challenge-invocation.js`; nothing in this scope imports back into it except `kairos-challenge-display.ts` importing `GENESIS_HASH` from `kairos-genesis-proof-hash.ts`.
