Two entry points form the module: `export.ts` registers the MCP `export` tool and dispatches on `input.format` into four branches — flat `markdown` (delegates to `dump.ts`), `trace_jsonl`/`reward_jsonl`/`sft_jsonl`/`preference_jsonl` (pulls pairs from `executionTraceStore` and serializes via `export-reward-jsonl`), `skill_tree` (in-memory JSON tree), and `skill_zip` (builds a manifest + either inline base64 zip or a deferred download via `mintExportDownloadCapability`). The `skill_zip` path composes the `skill-export/` sub-package: `collectSkillExportItemsForZip` (from `export-skill-items.js`, outside scope) yields `SkillExportItem[]`; each item's files are assembled by `assemble-skill-item.ts`, whose SKILL.md is produced by `build-skill-md.ts` using YAML frontmatter derived in `derive-metadata.ts`. Files are then passed through `artifact-sanitization/` (rule-based pipeline exported from `index.ts`) before being zipped by `zip-bundle.ts` (archiver, configurable compression level). `dump.ts` is a standalone layer-dump utility used both by this export flow and the HTTP API; it normalizes legacy/new `kairos://layer/{uuid}` URIs, optionally walks an adapter chain via `qdrantService.getAdapterLayers`, and renders protocol markdown with optional reward sections and contract fences. SHA-256 digests for artifacts come from `sha256.ts` / `sha256sums.ts`. Dependency direction is one-way: `export.ts` → `dump.ts` + `skill-export/*`; `skill-export/artifact-sanitization/*` has no inward deps beyond its own types.
