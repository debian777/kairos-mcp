Three small, single-responsibility modules consumed by the broader tools layer:
- `artifact-catalog.ts` exposes `listAdapterArtifacts`, which paginates through a Qdrant collection via `MemoryQdrantStore.getQdrantAccess()`, filters points with `buildSpaceFilter` (tenant-scoped) plus an `adapter.id` + `content_type` must clause, maps raw payload into the typed `AdapterArtifactMetadata` interface, and sorts by name then UUID.
- `artifact-mime.ts` is the canonical MIME allowlist: it aggregates `EXTENSION_TO_ALLOWED_MIMES` and `NO_EXTENSION_ALLOWED_MIMES` from `./skill-export/artifact-sanitization/extension-mime-map.js` into a frozen `ALLOWED_ARTIFACT_MIMES` array plus a Set-backed `isAllowedArtifactMime` checker, and provides `inferArtifactMimeFromName` using Node's `path.extname`.
- `artifact-relative-path.ts` provides `normalizeArtifactRelativePath`, a pure validator that rejects absolute paths, `..` segments, empty segments, trailing slashes, backslashes, and anything over 2048 chars, returning a forward-slash-normalized path or null.
Dependency direction is one-way: catalog depends on mime and tenant-context; mime depends only on the extension-mime map; relative-path has no internal deps. No runtime side effects beyond module-load-time Set/Map construction.
