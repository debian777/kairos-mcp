Three independent, side-effect-free utilities consumed by the MCP tool layer:
- `mcp-contract-match.ts` implements subset-based matching of `solution.evidence.arguments` against a `ProofOfWorkDefinition.mcp` contract (plain-object keys are required subsets, arrays must be fixed-length with per-index recursion, primitives use SameValue/NaN-equality) and exposes `validateMcpSubmissionAgainstContract`, which returns a discriminated union `{ ok: true } | { ok: false; code: 'MCP_TOOL_MISMATCH' | 'MCP_ARGUMENTS_MISMATCH' | 'MISSING_FIELD'; message }`.
- `mcp-loose-input-schema.ts` provides `mcpLooseToolInput<T>(strict)` — a Zod factory that unions the strict schema with `z.record(z.string(), z.unknown()) | undefined | null` so the MCP SDK's pre-validation never rejects a call before the handler can `safeParse` and reply with an agent-teaching body.
- `mcp-runtime-error.ts` wraps `EmbeddingRateLimitError` into the MCP error-result shape (`{ isError: true; content: [{ type: 'text'; text: JSON.stringify({...}) }] }`) with a human-readable `retry_hint`, returning `null` for non-rate-limit errors so callers can re-throw them unchanged.
Dependency direction is one-way: this module imports only from `../types/memory.js` and `../services/embedding/providers.js`; nothing in this module is imported by other leaf modules except the broader tools layer.
