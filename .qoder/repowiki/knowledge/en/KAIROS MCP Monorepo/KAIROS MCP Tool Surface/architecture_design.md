Each top-level `*.ts` file is a self-contained MCP tool: it owns its Zod input/output schema (in a sibling `_schema.ts`), an `execute*` function that performs business logic against `MemoryQdrantStore`/`QdrantService`, and a `register*Tool` wrapper that parses input via Zod, emits OpenTelemetry metrics (`mcpToolCalls/mcpToolDuration/mcpToolErrors/mcpToolInputSize/mcpToolOutputSize`), formats errors through `forward-tool-error` / `mcp-runtime-error`, and returns `{ content, structuredContent }`. Cross-cutting concerns are factored into sibling modules so each tool stays thin:
- Protocol identity: `kairos-uri.ts` + `kairos-challenge-display.ts` + `kairos-genesis-proof-hash.ts` provide canonical URI parsing/building, genesis hash, and challenge-shape display used by activate/forward/reward/export.
- Proof-of-work orchestration: `next-pow-helpers.ts` validates submissions; `next-missing-proof-payload.ts` builds MISSING_PROOF payloads; `next-previous-step.ts` routes to prior steps; `next.ts` updates step quality. `forward.ts` composes these with `forward-view.ts` (interactive contract view) and `forward-trace.ts` (tensor trace persistence).
- Export pipeline: `export.ts` dispatches on `format` to `dump.ts` (markdown), `export-source.ts` (raw source), `export-reward-jsonl.ts` (training JSONL), or `skill-export/` (tree/zip with sanitization rules in `artifact-sanitization/`).
- Train/tune/update: `train.ts`/`tune.ts`/`update.ts` share `train-store.ts`, `tune-execute.ts`, `tune-messages.ts`, `tune-verify.ts`, and `misc_tools` delete/review helpers.
- Shared contracts: `schemas/` holds reusable Zod fragments; `mcp-contract-match.ts`, `mcp-loose-input-schema.ts`, `mcp-tool-input-teaching.ts`, `mcp-runtime-error.ts` standardize validation and teaching-friendly error responses across tools.
- Artifact safety: `artifact-catalog.ts`, `artifact-mime.ts`, `artifact-relative-path.ts`, `local-artifact-dir-contract.ts` enforce MIME allowlists and traversal-safe paths consumed by activate's linked-artifact materialization commands and skill-export sanitization.
All tools depend on `../services/*` (memory store, Qdrant, execution-trace-store, proof-of-work-store, forward-runtime-store) and `../utils/*` (tenant context, structured logger, space display); none import each other directly except for the register→execute split within the same tool.
