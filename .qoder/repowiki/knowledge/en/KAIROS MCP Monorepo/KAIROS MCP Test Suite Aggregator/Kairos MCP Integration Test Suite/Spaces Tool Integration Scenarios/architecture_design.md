Each file is a thin scenario shim: it imports `runSpacesToolContract` from `../contracts/spaces-tool.contract.js` and a transport-specific harness (`http-simple`, `http-auth`, `stdio-simple`) from `../harness/`, then conditionally executes or skips the suite based on runtime flags — `isHttpTransport()` / `serverRequiresAuth()` from `../../utils/auth-headers.js` for HTTP variants, and `process.env.ENV === 'dev_stdio'` for the stdio variant. The actual test logic lives in the contract; these files only select which harness to plug in, keeping transport selection orthogonal from assertion logic.
