Each `.test.ts` file targets one surface of the running server: `http-api-*.ts` files cover `/api/train/raw`, `/api/forward`, `/api/reward`, `/api/snapshot`, `/api/activate`, dump/activate flows, slug forwarding, route errors, training JSON/similarity/space validation; `auth-keycloak.test.ts` validates 401-with-login_url when unauthenticated and structured `/api/me` groups when authenticated; `metrics-*.ts` hit `/api/metrics`. All tests share `http-api-test-helpers.ts`, which centralizes `BASE_URL` / `API_BASE`, an `apiFetch` wrapper that injects auth headers via `getAuthHeaders()`, and the pre-encoded `REVIEW_EVIDENCE_PARAM` query string built from `MOCK_REVIEW_EVIDENCE`. Tests are gated by `isHttpTransport()` so they skip in non-HTTP transports. The suite expects a running server (started via `npm run dev:test`) and optional Testcontainers Keycloak (`KEYCLOAK_URL= AUTH_ENABLED=true`).
