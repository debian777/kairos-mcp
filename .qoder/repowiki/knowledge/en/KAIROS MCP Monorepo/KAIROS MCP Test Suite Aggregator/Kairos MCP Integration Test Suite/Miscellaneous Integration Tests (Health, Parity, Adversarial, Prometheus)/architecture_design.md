Each file is an independent Jest test suite targeting a single concern; there is no shared orchestration layer inside this module — suites import helpers from `tests/integration/utils/` (`mcp-client-utils`, `auth-headers`, `expect-with-raw`) and from `src/tools/*_schema.js` for Zod-based output validation. The `utils/adapter-space-mcp-context.ts` bundle provides a reusable MCP + spaces snapshot plus skip/reason logic used by `adapter-space-fork.test.ts`, `adapter-space-move.test.ts`, and `space-crud-per-space.test.ts`. All suites gate themselves on transport via `isHttpTransport()` so they are skipped when the runner uses the non-HTTP transport. Dependency direction is one-way: test files → shared utils → source schemas; nothing in this module is imported by production code.
