Flat collection of Jest test files under `tests/integration/`, each file owning one scenario group (`describe`) for a single tool or behavior: `kairos-train-*.test.ts` covers the `train` tool (basic happy path, duplicate/similarity policy, AI_CODING_RULES import, docs examples, edge cases, validation, heading sanitization, artifact handling); `kairos-search-*.test.ts` covers the `activate` tool (cases 1–4 from reports/outputs.md, forbidden behaviors, perfect matches, scores, access control); `kairos-dump.test.ts` validates the `export` tool; `kairos-qdrant-storage.test.ts` reaches into Qdrant directly to assert vector persistence. All tests share a common bootstrap via `../utils/mcp-client-utils.js` (`createMcpConnection`) and assertion helpers in `../utils/expect-with-raw.js` (`parseMcpJson`, `withRawOnFail`). External state is injected through `MOCK_REVIEW_EVIDENCE` so no real LLM review is needed. The suite depends on an external Qdrant instance (URL/collection/api-key from env) and a running Kairos MCP server reachable by the shared client.
