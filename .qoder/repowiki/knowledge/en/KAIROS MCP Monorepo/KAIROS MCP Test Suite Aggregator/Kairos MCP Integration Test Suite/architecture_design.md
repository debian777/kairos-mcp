All children share a single transport-agnostic harness layer under `harness/` that defines the `TestHarness` interface (`scenario`, `callTool`, `close`) and three concrete implementations — `http-simple.ts`, `http-auth.ts`, `stdio-simple.ts` — each guarded by runtime checks on `AUTH_ENABLED`, `ENV`, and embedding config. The `contracts/` directory holds transport-neutral contract suites (e.g. `spaces-tool.contract.ts`) parameterised by a `createHarness` factory, while per-transport scenario entry points in `scenarios/` wire a specific harness into those contracts. Shared utilities (`utils/mcp-client-utils.js`, `utils/auth-headers.js`, `utils/adapter-space-mcp-context.ts`) plus the `mcp-call-normalize` helper provide the common client bootstrap, auth header injection, and JSON-RPC normalisation used by every child module. Each child scope targets one surface area (HTTP REST API, MCP protocol, CLI commands, Kairos train/search/reward, v4 forward, MIME/skill export parity, spaces scenarios, misc cross-cutting) but they all converge on the same `TestHarness` + `SCENARIOS` registry so the same assertions run over multiple transports.
