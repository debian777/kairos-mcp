Each `.test.ts` file is a standalone Jest suite that spawns the compiled CLI binary (`dist/cli/index.js`) via `child_process.exec/execFile`, so tests drive the CLI through its actual argument parser rather than importing it. `cli-commands-shared.ts` centralises the shared harness: `setupCliConfigWithLogin()` writes a bearer token into the XDG config directory (mimicking `src/cli/config-file.ts`'s path resolution), `execAsync`/`execAsyncNoAuth`/`execAsyncWithConfig` wrap command execution with consistent env (`BROWSER=true`, `KAIROS_REVIEW_EVIDENCE`), and `requireMcpServerAndCliLogin` turns silent skips into hard failures. Auth-related suites (`cli-auth-browser-login.e2e.test.ts`, `cli-commands-env.test.ts`) gate themselves on `serverRequiresAuth() && isHttpTransport()`; non-auth suites use `isHttpTransport() ? describe : describe.skip`. `stdio-launch-smoke.test.ts` is separate — it spawns `dist/bootstrap.js` (or `src/bootstrap.ts` via ts-node) over stdio and drives it with `@modelcontextprotocol/sdk`'s `StdioClientTransport`, asserting no stray stdout bytes before the MCP handshake. Test data lives under `tests/test-data/cli-minimal-test.md` and is created/removed in temp dirs per test.
