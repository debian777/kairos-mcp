Six Jest test files form one cohesive suite around the `skill_export` endpoint. A shared runtime helper (`skill-export-shared.ts`) centralises adapter/artifact training via `/api/train`, export calls to `/api/export`, and response helpers for `ExportSkillTreeResponse` / `ExportSkillZipResponse`. Three transport-specific suites exercise the same fixture pipeline:
- `mime-fixture-export-api.test.ts` — HTTP fetch against the live server.
- `mime-fixture-export-cli.test.ts` — pre-trains via API then drives `node kairos-mcp-docs/bin/kairos.js export` subprocesses.
- `mime-fixture-export-mcp.test.ts` — uses `createMcpConnection()` + `callTool({ name: 'train'|'export', ... })`.
Each transport follows an identical Stage 0 → Stage 1 → Stage 2 loop: train the MIME fixture, export, clean up, retrain from the exported bundle, export again, and assert SHA256SUMS body identity between stages. The multi/coverage suites (`skill-export-multi.test.ts`, `skill-export-coverage.test.ts`) cover single-adapter shape, artifact-in-ZIP layout, multi-adapters via `adapters[]`, and `all_adapters + space_name` on a group space. All assertions delegate to `../utils/skill-bundle-sha-assert.js` (SHA row parsing, self-verification) and `../utils/mime-artifact-fixture-contract.js` (fixture manifest), with dumps written via `../utils/mime-fixture-export-dump.js` under a per-transport/format/stage directory tree.
