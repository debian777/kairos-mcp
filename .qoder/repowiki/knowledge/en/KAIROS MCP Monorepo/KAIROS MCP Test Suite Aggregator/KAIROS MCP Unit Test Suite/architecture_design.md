Flat collection of Jest test files under `tests/unit/`, each file targeting a single source module or feature area (e.g. `adapter-builder.test.ts` → `src/services/memory/adapter-builder.js`, `cli-config-file.test.ts` → `src/cli/config-file.js`). Tests import production code via relative paths into `../../src/...` and assert on exported functions; there is no shared test harness inside this directory beyond the global `tests/setup.ts` which seeds `QDRANT_URL` and refreshes auth tokens. External side effects are isolated through `jest.unstable_mockModule(...)` calls at the top of each test file (keyring, output, etc.), so individual tests remain deterministic without Docker. A custom Jest sequencer (`tests/jest-sequencer.cjs`) lives alongside but only affects integration suites; unit tests run in parallel by default. The suite uses `@jest/globals` for `describe`/`test`/`expect`, `ts-jest` for TypeScript execution, and `zod` `.safeParse()` results as the primary assertion surface across schema-focused tests.
