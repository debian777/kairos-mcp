Four cooperating sub-packages with one-way dependencies (tests/workflow-test → tests/utils; tests/utils → Node stdlib + testcontainers; no reverse imports):
- `tests/mocks/` — Jest runtime stubs loaded via `moduleNameMapper`, e.g. `uuid-stub.cjs` which returns deterministic v4/v5 UUIDs so tenant-space IDs stay stable across runs.
- `tests/test-data/` — immutable fixture trees consumed by tests. `mime-artifact-sample/` and `mime-artifact-sample-external/` are skill bundles whose contents are declared in `artifact-contract.json` (paths, mime types, expected slugs) and validated at load time by `utils/mime-artifact-fixture-contract.ts`; `SHA256SUMS` anchors content integrity.
- `tests/utils/` — pure helpers split by concern: environment/bootstrap (`keycloak-container.ts`, `keycloak-existing-env.ts`, `auth-headers.ts`, `keycloak-client-admin.ts`), data/format parsers (`zip-parser.ts`, `prometheus-parser.ts`, `mcp-list-tools-schema-helpers.ts`), assertion aids (`expect-with-raw.ts`, `mock-review-evidence.ts`, `skill-bundle-sha-assert.ts`, `adapter-space-test-helpers.ts`), and the MIME export parity pipeline (`mime-artifact-fixture-contract.ts`, `mime-fixture-export-dump.ts`, `artifact-fixture-cleanup.ts`). The Keycloak module exposes two bootstraps — `startKeycloakWithTestUser()` spinning a `quay.io/keycloak/keycloak:26.0` container via testcontainers, or `useExistingKeycloakFromEnv` reusing an external instance — both returning a `KeycloakTestEnv` with a `getTestUserToken` factory that defaults to the `openid kairos-groups` scope (overridable via `KAIROS_TEST_OIDC_SCOPE`).
- `tests/workflow-test/` — a small eval harness (`eval-harness.ts`) defining `EvalCaseResult` / `EvalSuiteResult` shapes plus `runEvalSuite` and `getEvalFailures`, used by `tests/integration/reward-*evals.test.ts` to drive code-graded scenarios against the dev server; `PROMPT.md` is an optional rubric prompt for a secondary evaluator model after automated checks pass.
