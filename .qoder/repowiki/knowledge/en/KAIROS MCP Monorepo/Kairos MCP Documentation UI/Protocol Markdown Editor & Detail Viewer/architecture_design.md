Two sibling route-level components compose the feature:
- `ProtocolDetailPage.tsx` is read-only: it fetches a protocol via `useProtocol`, parses it with `parseProtocolMarkdown`, and renders a step flow graph, challenge cards, triggers/completion sections, and export actions (markdown download, skill zip bundle).
- `ProtocolEditPage.tsx` is the editor: it owns the full `ProtocolFormState` in local state, hydrates it from an existing protocol or an uploaded `.md` file via `parseProtocolMarkdownToForm`, and persists by calling `/api/train` (new) or `/api/tune` (existing), optionally moving between spaces or forking from a `kairos://adapter/...` URI.

The editor delegates UI to four sub-components under `protocol-edit/`: `ProtocolEditSpaceSection` (space/fork selection), `ProtocolEditImportSection` (file upload + link to skill editor), `ProtocolEditStepCard` (per-step label/type/body/challenge fields), and `ProtocolEditPreviewColumn` (live parsed preview). Shared constants (`DEFAULT_STEP`, `CHALLENGE_TYPE_KEYS`, `SOURCE_ADAPTER_URI_RE`) and the Zod schema `markdownSchema` live in `constants.ts`. Dependency direction is one-way: page → sub-components → shared `@/components/*` and `@/hooks/*`; no cross-imports between detail and edit pages.
