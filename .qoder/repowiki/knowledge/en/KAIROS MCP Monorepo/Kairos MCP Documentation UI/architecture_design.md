The `src/ui` tree is a single Vite/React application split by feature area rather than by layer: `App.tsx` + `main.tsx` (app_shell) mount providers (i18next, React Query, theme tokens) and declare lazy-loaded routes under `/ui`; each feature folder (`pages/*`, `components/run`, `protocol-edit`, `kairos-page-sections`) owns its own route, components, and local state. Cross-cutting concerns live in sibling folders consumed uniformly by every page — `hooks/` (auth, spaces, runs, protocol, reward), `lib/` (JSON-API client, skill ZIP export, TipTap extensions), `utils/` (browse adapters, confidence formatting), `i18n/` (English keys), and `theme/` (CSS custom properties for light/dark). Pages compose reusable shell/markdown pieces from `components/` (Layout, RenderedMarkdown, RichTextEditor, SurfaceCard, CopyButton, ErrorAlert) and pull data exclusively through the hooks, so no page reaches into another feature's internals.
