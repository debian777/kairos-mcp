Three sibling React components form a single route surface:
- `KairosPage.tsx` is the top-level page component. It owns all query/space state, syncs it with `react-router-dom`'s `useSearchParams`, and orchestrates two data hooks — `useActivate(submittedQuery, submittedSpace)` for search results and `useSpaces(true, { includeAdapterTitles: showBrowse })` for the A–Z browse view. It renders either `KairosBrowseByLabelSection` (when no query) or `KairosActivateResultsSection` (when choices are returned), plus shared loading/error skeletons.
- `kairos-page-sections.tsx` holds two presentational sub-components (`KairosBrowseByLabelSection`, `KairosActivateResultsSection`) that receive precomputed props; they contain no data fetching and depend only on `@/components/SpaceSelect`, `@/utils/browse-adapters`, `@/utils/confidence`, and the `ActivateOutput` type from `tools/activate_schema.js`.
- `NotFoundPage.tsx` is a minimal fallback rendered for unmatched routes, delegating to `ErrorAlert`.
Dependency direction is strictly inward: pages → shared hooks/components/utils; sections have no side effects. Styling uses Tailwind classes bound to CSS custom properties (e.g. `--color-primary`, `--layout-touch-target`).
