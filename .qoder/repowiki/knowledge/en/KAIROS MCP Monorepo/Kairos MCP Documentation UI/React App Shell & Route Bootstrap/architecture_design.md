Three-file leaf that wires together the entire UI runtime:
- `main.tsx` is the browser entry point: it initializes a single `QueryClient`, then renders `<StrictMode>` with nested providers — `QueryClientProvider`, `ThemeProvider` (from `./hooks/useThemePreference`), and finally `App`. It also imports `./i18n` for side-effect initialization and `./index.css` for global styles.
- `App.tsx` owns routing via `react-router-dom`: `BrowserRouter` is configured with `basename="/ui"` so the SPA lives under a sub-path. The default export `App` wraps an exported `AppRoutes` component, which declares all routes (`/`, `/kairos`, `/runs`, `/protocols/*`, `/account`, `*`) using `lazy()` + dynamic `import()` per page module plus a shared `<Suspense fallback={<RouteFallback />} />` pattern to keep the initial chunk small. Every route is rendered inside a shared `<Layout />` shell component from `./components/Layout`.
- `index.css` is the single CSS entry: it imports design tokens from `./theme/tokens.css`, registers Tailwind v4's `@import "tailwindcss"`, and maps every token into Tailwind's `@theme inline` namespace so components use semantic names like `text-[var(--color-text)]` rather than raw values. It also defines global accessibility utilities (`.skip-link`, `.sr-only`) and a `prefers-reduced-motion` reset.

Dependency direction is strictly inward: this leaf depends on sibling modules (`pages/*`, `components/Layout`, `hooks/useThemePreference`, `theme/tokens.css`, `i18n`) but exposes no internal APIs beyond the `AppRoutes` export used by tests.
