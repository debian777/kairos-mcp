Three cooperating layers with strict one-way dependency:
- Bootstrap (`index.ts` → `program.ts`) creates the root Commander instance, reads version from `../../package.json`, attaches global flags (`--url`, `--timeout`, `--retries`, `--no-browser`), and uses a `preAction` hook to propagate them into `KAIROS_API_URL` / `KAIROS_NO_BROWSER` env vars so downstream code can read them uniformly.
- Command registry (`program.ts`) imports each subcommand's factory function from `commands/*.ts` (search, begin/train, spaces, update/delete, attest/reward, export, serve, login/logout/token) and calls it with the parent program to attach the subcommand tree.
- Shared runtime (`client-factory.ts`, `config-file.ts`, `keyring.ts`, `oauth-refresh.ts`, `output.ts`, `safe-http-url.ts`, `resolve-api-base.ts`, `rewrite-login-url.ts`, `download-export-ref.ts`, `skill-zip-local-write.ts`, `upload-guards.ts`, `auth-error.ts`) provides the HTTP client, XDG config store, keyring-backed credential persistence, OAuth refresh flow, and output helpers that every command consumes via `createClientFromProgram` / `resolveClientOptions` re-exported by `program.ts`.

Cross-child wiring: commands never import each other; they depend only on the shared runtime layer, while `program.ts` is the single place where the command surface is composed. Global options are the only cross-cutting contract between the bootstrap layer and individual commands.
