Three cooperating modules form the shared runtime consumed by all commands:
- `api-client.ts` exports the `ApiClient` class. Constructor resolves `baseUrl` via explicit arg → `KAIROS_API_URL` env → `getDefaultApiUrlFromFile()` → build-time default from `./config.js`, then delegates all HTTP to a private `request<T>()` that centralises timeout (`AbortController`), exponential-backoff retries on transient network errors (ECONNREFUSED/ECONNRESET/EPIPE/ENOTFOUND/UND_ERR_CONNECT_TIMEOUT + AbortError), 429 `Retry-After` backoff, proactive JWT refresh within `PROACTIVE_REFRESH_SKEW_SEC`, singleflighted `refresh_token` exchange via `runRefreshWithSingleflight`, and 401-driven login-or-refresh retry chains. Thin typed wrappers (`activate`, `forward`, `train`, `tune`, `trainJson`, `delete`, `spaces`, `reward`, `export`, `downloadExportRef`) expose one endpoint each.
- `client-factory.ts` is the only place Commander `Command` objects are consumed: `resolveClientOptions(program)` reads global flags `--timeout` / `--retries` (precedence flag > env var > default) and `getResolvedApiBaseFromProgram`, then `createClientFromProgram` constructs an `ApiClient`. This isolates program parsing from the HTTP layer to avoid circular imports with command modules.
- `config-file.ts` owns persistence of tokens and per-environment URLs in an XDG directory. It supports two shapes — legacy flat `{ KAIROS_API_URL, bearerToken }` and new `{ defaultUrl, environments: { [url]: { bearerToken, refreshToken } } }` — migrating inline secrets into the OS keyring (via `./keyring.js`) and leaving `KEYCHAIN_TOKEN_PLACEHOLDER` markers in the file. `readConfig(baseUrl?)` returns a `CliConfig` whose tokens are resolved from keyring first, falling back to file; `getDefaultApiUrlFromFile()` provides a synchronous read without touching the keyring for use during `ApiClient` construction.

Dependency direction inside the module: `client-factory.ts` → `api-client.ts` → `config-file.ts`; `api-client.ts` additionally depends on `auth-error`, `oauth-refresh`, `commands/login`, `rewrite-login-url`, `upload-guards`, and `download-export-ref` for cross-cutting concerns, while `config-file.ts` delegates I/O to `config-file-internals` and `config-file-write` submodules.
