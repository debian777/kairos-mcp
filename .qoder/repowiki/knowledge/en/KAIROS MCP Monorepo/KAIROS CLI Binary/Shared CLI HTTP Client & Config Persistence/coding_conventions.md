- Configuration precedence follows a fixed order: explicit constructor/flag argument > process.env variable > persisted config file > compile-time default, applied uniformly to baseUrl, timeoutMs, and maxRetries.
- Transient failures are retried with exponential backoff capped at 10s plus jitter, logged to stderr with a `[kairos] retry N/M:` prefix, and distinguished from fatal errors by checking `err.name === 'AbortError'` or known `E*` codes.
- HTTP error handling is centralized in `request<T>()`: 401 triggers a singleflight refresh attempt before interactive login, 429 honours `Retry-After`, and `SPACE_NOT_FOUND`/`SPACE_READ_ONLY` responses are augmented with the server's `available_spaces` list.
- Secrets are never stored inline when a keyring is available: `readConfig` migrates inline tokens into the keyring and replaces them with `KEYCHAIN_TOKEN_PLACEHOLDER` entries in the config file via `persistEnvironmentSentinel`.
- Each public API method is a thin wrapper around `this.request<T>(endpoint, options)` that serializes its arguments into JSON or sets specific headers (e.g. `x-space`, `x-force-update`, `x-review-evidence`), keeping transport details out of callers.
