Three sibling Commander command modules under `src/cli/commands/`. `login.ts` is the authority: it exports `getBaseUrl`, `isTokenValid`, and `loginWithBrowser` which `logout.ts` and `token.ts` import to share base-URL resolution, token validation, and the PKCE callback server. The PKCE flow in `loginWithBrowser` spins up a local HTTP server on `127.0.0.1` (port configurable via `KAIROS_LOGIN_CALLBACK_PORT`) with a per-request random path token to prevent shared-link attacks, exchanges the authorization code at the OIDC token endpoint discovered from `/.well-known/oauth-protected-resource`, and persists `{ apiUrl, bearerToken, refreshToken }` through `writeConfig` in `config-file.js`. `logout.ts` clears credentials by writing `bearerToken: null`; `token.ts` prints the stored token to stdout for scripting and optionally re-validates or triggers login. All I/O goes through `../output.js` (`writeStdout`/`writeError`) so tests can capture output; process exit codes are used as the only error signal.
