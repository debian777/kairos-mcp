Five independent CLI entry points under `scripts/`, each a self-contained script invoked directly (via `ts-node` for `.ts` or `node` for `.mjs`) rather than forming a shared library.

- `build-embed-docs.ts` is the primary generator: it walks `src/embed-docs/{prompts,tools,resources,templates,mem}` and emits `src/resources/embedded-mcp-resources.ts`, exporting an `mcpResources` object plus typed getters (`getPrompt`, `getResource`, `getMetaDoc`, `getTemplate`, `getToolDoc`, `listResourceKeys`). It delegates YAML frontmatter slug extraction to the sibling helper `build-embed-docs-slug-meta.ts` (`extractSlugFromMarkdown`, `collectMetaBySlug`) and additionally copies `mem/*.md` into `dist/embed-docs/mem/` so the runtime can read them from disk.
- `build-vite-ui-env-define.ts` is a small pure-function module exposing `getPackageVersionForUi()` and `getUiImportMetaEnvDefine()`, consumed by Vite/Vitest config to bake `import.meta.env.VITE_KAIROS_VERSION` from repo root `package.json`.
- `build-sync-skill-versions.mjs` is a dual-mode (update vs `--check`) synchronizer: it reads `package.json` version as the mem target, resolves the last stable git tag via `git tag -l "v*"` as the skills target, then rewrites `skills/*/SKILL.md` metadata.version and `references/KAIROS.md` frontmatter version fields using regex-based line replacement; mem files in `src/embed-docs/mem/*.md` are updated against the package version instead of the last stable tag.
- `build-ensure-tgz.mjs` is an idempotent gate that runs `npm run build` + `npm pack --pack-destination dist` only when `dist/debian777-kairos-mcp-<version>.tgz` is missing.

Dependency direction is one-way: helpers are imported by generators; nothing in this module imports application code. There is no shared runtime — all side effects happen at build time.
