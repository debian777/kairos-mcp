Flat collection of independent CLI entry points under `scripts/`, each self-contained with no cross-imports:
- Node.js ESM tools (`*.mjs`) — `journey-export.mjs` streams audit JSONL into per-correlation journey files; `journey-diff.mjs` compares two journey directories reporting tool-sequence/response-shape/error-rate/timing drift; `journey-replay.mjs` replays a journey via MCP JSON-RPC (`tools/list` + `tools/call` over `/mcp`) with optional strict CI exit codes; `dev-mime-fixture-train-export.mjs` trains MIME fixtures through the server's `/api/train/raw` + `/api/train` endpoints and writes back `SKILL.md`/`SHA256SUMS` from the exported skill tree.
- Python linter `lint-agent-skills.py` that auto-provisions an in-repo `.venv`, installs `skills-ref` from GitHub, validates every `skills/<dir>/SKILL.md` (plus `skills/.system/`), and rejects relative markdown links.
- TypeScript AST scanner `lint-verify-clean-source.mjs` walks `src/**` and `tests/integration/**` to ban `console.*`, mock imports/calls/members, and `__mocks__`/`mocks/` directories.
- Bash helpers: `qdrant-binary.sh` manages a local Qdrant binary under `.local/bin` (install/start/stop/status with pidfile + log); `dev-node-use.sh` writes `.nvmrc` and activates via `fnm`; `stdio/entrypoint.sh` is a container entrypoint that boots Qdrant, waits on `/readyz`, then execs the built MCP bootstrap.
- `kairos-db-init/README.md` documents the deprecated Postgres init script path.
Dependency direction is one-way outward: scripts read repo state or call the live server; nothing in this module is imported by production code.
