Flat collection of standalone CLI entry points under `scripts/`, each self-contained with no shared library — callers invoke them directly from npm scripts or CI. Two functional clusters exist:
- Documentation tooling: `fix-docs-protocol-wording.py` applies a fixed ordered substitution table (`SUBS`) across `docs/**`, `skills/**`, `src/mcp-apps/**`, `scripts/**`, and the root `README.md`, skipping Keycloak `/protocol/` URLs and an allowlisted file; `validate-mermaid.sh` extracts fenced mermaid blocks via awk and validates them through `@mermaid-js/mermaid-cli` (`mmdc`).
- Wiki publishing: `setup-github-wiki-permissions.sh` uses the `gh` CLI to enable the wiki feature and set Actions workflow permissions to write; `sync-wiki.sh` clones/pulls `kairos-mcp.wiki.git` into `/tmp/<repo>.wiki`, mirrors `.qoder/repowiki/en/content/` via `rsync -a --delete`, then commits and pushes.
- Test snapshot lifecycle: `seed-test-snapshot.sh` trains adapters against a running KAIROS app (via the built CLI at `dist/cli/index.js train`) and captures Qdrant snapshots for both the main collection and `_traces`; `import-test-snapshot.sh` drops existing collections, restores snapshots via Qdrant's `/collections/{name}/snapshots/upload` endpoint, and verifies point counts — in CI it auto-invokes the seed script when the cache is missing.
