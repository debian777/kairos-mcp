- Each harness is a self-contained CLI with a shebang `#!/usr/bin/env node`, top-level JSDoc usage block, and an `async main()` invoked via `main().catch(...)` exiting with code 1 on error.
- Base URL resolution is centralized in `test-integration-app-base-url.mjs` using the precedence `KAIROS_BASE_URL` > `KAIROS_API_URL` > `http://localhost:$SERVER_PORT` (default 3300); other scripts import `resolveKairosAppBaseUrl` rather than reimplementing it.
- Bearer authentication is optional and loaded once via `loadIntegrationBearer(root)`, falling back from `KAIROS_INTEGRATION_BEARER` env to `.test-auth-env.dev.json`, then injected through `buildAuthHeaders(bearer)` as an `Authorization` header only when present.
- HTTP responses are never persisted raw: `proof-utils` converts them into typed proof objects (`buildTrainResponseProof`, `buildActivateResponseProof`, `buildForwardResponseProof`, `buildRewardResponseProof`) and `report-utils` runs them through `snapshotForIntegrationReport` which redacts JWTs, Bearer tokens, URLs, and caps depth/array/key counts before writing.
- Report output is guarded against path traversal and size blow-up: `writeIntegrationReportFile` resolves paths relative to `reports/` and truncates content to `MAX_REPORT_BYTES` (4 MiB) with a trailing marker.
