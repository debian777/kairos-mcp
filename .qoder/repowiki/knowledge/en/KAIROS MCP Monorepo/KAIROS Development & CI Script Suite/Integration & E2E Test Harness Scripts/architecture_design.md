A flat collection of executable Node (`.mjs`) and one Bash script under `scripts/`, each a standalone CLI invoked directly via `node` or `sh`. The AI-MCP integration suite is layered into four cooperating modules: `test-ai-mcp-integration.mjs` is the orchestrator that iterates `docs/examples/*.md` adapters through the Train → Activate → Forward loop → Reward lifecycle against `/api/train/raw`, `/api/activate`, `/api/forward`, `/api/reward`; `test-ai-mcp-integration-auth-utils.mjs` loads the bearer token from `KAIROS_INTEGRATION_BEARER` or `.test-auth-env.dev.json`; `test-ai-mcp-integration-proof-utils.mjs` classifies URIs, contracts, choices and builds normalized request/response proof objects; `test-ai-mcp-integration-report-utils.mjs` sanitizes/redacts secrets (JWTs, Bearer tokens, URLs) and writes bounded markdown reports under `reports/<RUN_ID>/...`. All scripts share `test-integration-app-base-url.mjs` for base-URL resolution (`KAIROS_BASE_URL` > `KAIROS_API_URL` > `http://localhost:$SERVER_PORT`, defaulting to port 3300 matching `deploy-run-env.sh`). Independent helpers include `test-capture-viewports.mjs` (Playwright headless Chromium screenshots at mobile/tablet/desktop/wide), `test-embedding-key.mjs` (OpenAI `/v1/embeddings` reachability check), and `test-helm.sh` (chart-testing + kubeconform + helm-unittest pipeline). Dependency direction is strictly inward toward these shared utils — no cross-import between sibling harnesses.
