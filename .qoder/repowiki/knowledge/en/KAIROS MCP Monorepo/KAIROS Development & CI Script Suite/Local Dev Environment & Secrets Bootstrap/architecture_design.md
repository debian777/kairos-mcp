Single-purpose leaf scripts with no shared library — each file is an independent entry point invoked from npm scripts or directly:
- `scripts/env/.env.template` is the canonical fullstack variable catalogue; `scripts/deploy-generate-dev-secrets.py` parses it, resolves `__VAR__` placeholders against env/existing `.env`, fills missing secrets via `secrets.token_urlsafe/token_hex`, and writes repo-root `.env`. `scripts/env/create-env.sh` is a thin guard that calls the generator only when `.env` is absent.
- `scripts/deploy-copy-env-from-main.sh` uses `git worktree list` to locate the main worktree and copies its `.env*` files into the current worktree, so non-main branches share secrets without committing them.
- `scripts/deploy-run-env.sh` is the environment orchestrator: it loads `.env` plus optional `.env.<ENV>` profile overrides (`dev`, `dev_simple`, `dev_stdio`, `prod`), starts/stops the Node server (compiled `dist/bootstrap.js` or `src/bootstrap.ts` via ts-node) with `dotenv -e`, manages PID/log files under `.kairos-*.pid|log`, auto-starts the embedded Qdrant binary for `dev_simple`, runs post-start health checks against `/health`, and invokes `deploy-configure-keycloak-realms.py` to seed realms. It also wraps Jest for tests, forwarding extra args.
- `scripts/deploy-dev-cli-ready.sh` is a one-shot smoke test: build, configure Keycloak realms, perform browser login via `npm run cli:dev -- login`, then assert `search test` returns choices.
- `scripts/deploy-raw-qdrant-search.mjs` is a standalone Node probe that embeds a query via OpenAI embeddings and queries Qdrant's `/collections/{c}/points/search` directly, printing raw similarity scores.
Dependency direction is strictly outward: these scripts depend on the built app, Keycloak/Qdrant/Redis services, and external CLIs (`docker compose`, `redis-cli`, `lsof`, `jq`, `curl`); nothing in this module imports code from the application tree.
