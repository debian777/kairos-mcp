The scripts/ directory is a flat workspace of independent CLI tools rather than a single executable; each script targets one lifecycle stage and they are composed by GitHub Actions workflows and the local `dev-run-env.sh` entrypoint. Cross-cutting concerns are shared through conventions rather than a common library: scripts read secrets from `.env` (generated by `deploy-generate-dev-secrets.py`), target services at fixed compose hostnames (`keycloak`, `qdrant`, `app`), and use the same JSON realm fixtures under `scripts/keycloak/import/`. The children split by concern — env/bootstrap, Keycloak bootstrap, Helm version sync, docs/wiki maintenance, build-time embedding, CI hooks, and test harnesses — but all scripts are invoked directly from Makefile targets or Action steps with no central orchestrator.
