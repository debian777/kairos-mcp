Layered facade over `@qdrant/js-client-rest`:
- `connection.ts` — `QdrantConnection` owns the raw `QdrantClient`, TLS/CA cert setup, health checks, exponential-backoff reconnection, and exposes `executeWithReconnect()` which every operation wraps.
- `service.ts` — `QdrantService` is the public facade (plus a default singleton `qdrantService`) that delegates to feature modules; it also exposes safe getters for `client`, `collectionName`, `qdrantUrl`, `apiKey` so callers can reach into the underlying client without touching private fields.
- Feature modules are split by concern, each receiving a `QdrantConnection` and calling `conn.executeWithReconnect(...)`: `memory-store.ts` / `memory-updates.ts` (upsert/delete), `memory-retrieval.ts` (point fetch with space-scoped access), `search.ts` (vector similarity via `embeddingService` + `buildSpaceFilter`), `protocol.ts` (step lookup by slug/domain/type/task), `quality.ts` (retrieval counters, attest boost, implementation stats), `reward-propagation.ts`, `listing.ts`, `resources.ts`, `snapshots.ts`, `initialization.ts` (collection creation, payload index setup, alias lifecycle, `backfillSpaceId`).
- Shared cross-cutting helpers live in `utils.ts` (`sanitizeAndUpsert`, `parseBooleanEnv`), `types.ts` (shared upsert/result interfaces), `undici-compat.ts` (fetch shim for alias REST calls), and `index.ts` (re-exports).
- Dependency direction: feature modules → `connection.ts` → `@qdrant/js-client-rest`; they also depend on `../../utils/*` (tenant context, space filter, URI builder, structured logger) and sibling services (`../embedding/service.js`, `../redis-cache.js`, `../metrics/qdrant-metrics.js`, `../id-generator.js`). No intra-module circular imports — each file is a leaf module of this scope.
