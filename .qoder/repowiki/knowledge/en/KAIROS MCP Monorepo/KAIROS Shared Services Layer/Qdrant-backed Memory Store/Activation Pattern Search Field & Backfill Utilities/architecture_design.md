Three focused files with a clear dependency direction:
- `activation-pattern-payload.ts` is the pure normalization layer: it resolves canonical `activation_patterns` from payload vs. nested `adapter.activation_patterns`, strips top-level patterns, rewrites the adapter object into a canonical shape, and reports whether the payload changed.
- `activation-search-fields.ts` consumes normalized inputs to assemble the six text fields (`primaryDenseText`, `titleDenseText`, `activationPatternDenseText`, `sparseText`, plus per-field copies) used as embedding prompts; it also exposes a convenience wrapper over `memory-accessors.getAdapterName` / `getActivationPatterns` for the in-memory `Memory` type.
- `activation-search-backfill.ts` is the only file that touches external systems: it paginates the Qdrant collection via `QdrantClient.scroll`, computes which points need updates by comparing existing payload/vector fields against freshly built ones, batches title + activation-pattern embeddings through `embeddingService.generateBatchEmbeddings`, and upserts both vector names (from `qdrant-utils`) and the four `_text` payload columns.

No internal sub-packages — cohesion comes from shared function exports rather than a barrel index. The module depends outward on `utils/qdrant-utils`, `utils/tenant-context`, `utils/space-filter`, `services/embedding/service`, and `types/memory`, but has no inbound callers within this scope.
