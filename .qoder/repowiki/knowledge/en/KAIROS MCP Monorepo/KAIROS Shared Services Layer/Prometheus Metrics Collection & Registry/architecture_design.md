- Single shared `Registry` instance exported from `registry.ts`, configured with default labels `service='kairos'`, `kairos_version` (from `utils/build-version`), and `instance` (env `INSTANCE_ID` or hostname); tenant isolation is enforced by requiring per-metric `tenant_id` labels rather than a global default.
- Each domain lives in its own file (`http-metrics.ts`, `system-metrics.ts`, `agent-metrics.ts`, `embedding-metrics.ts`, `mcp-metrics.ts`, `qdrant-metrics.ts`, `memory-metrics.ts`, `export-metrics.ts`, `anomaly-metrics.ts`) that imports the shared `register` and constructs `prom-client` metric instances via the `registers: [register]` option so they auto-register at import time.
- `system-metrics.ts` additionally drives periodic collection via `setInterval` every 5 s to update uptime, process start time, and Node.js memory usage gauges.
- Dependency direction is strictly inward: metric files depend only on `./registry.js` and `prom-client`; nothing in this module depends on application services, keeping metrics decoupled from business logic.
