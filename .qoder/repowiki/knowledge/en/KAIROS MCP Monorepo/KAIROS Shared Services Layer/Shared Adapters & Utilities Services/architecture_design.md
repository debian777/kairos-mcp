Flat collection of independent single-file services under `src/services/`, each exposing a class or pure functions consumed by higher-level tools/adapters with no inter-dependencies among the five files.
- `activate-refinement-store.ts` wraps a singleton `ActivateRefinementStore` around the shared `keyValueStore` to track per-execution refine counts with a 1-hour TTL; all mutations are wrapped in try/catch that log via `structuredLogger` and return safe defaults (0).
- `adapter-layer-count.ts` provides pure helpers (`maxLayerIndexFromAdapterPoints`, `effectiveAdapterLayerCount`) reconciling Qdrant adapter-point payloads against an in-memory `layer_count`, preferring the payload when stale cache disagrees.
- `adapter-navigation.ts` exposes `resolveAdapterEntry/FirstLayer/NextLayer/PreviousLayer` which query `qdrantService.getAdapterLayers(adapterId)` to walk the adapter's layer graph and build `ResolvedAdapterLayer` entries plus `kairos://` URIs via `buildAdapterUri`.
- `code-block-processor.ts` is a stateless `CodeBlockProcessor` that tokenizes fenced markdown blocks into `CodeBlock`s, extracts language-specific identifiers via regex families, filters keywords, and can append them as `[CODE_IDENTIFIERS: ...]` for embedding searchability.
- `id-generator.ts` is a static-only `IDGenerator` using `uuid` v4/v5 and `crypto.randomUUID()` with a hard-coded `KAIROS_NAMESPACE` to derive deterministic Qdrant IDs from human URIs (`kairos://...`), supporting both bare UUIDs and layered step URIs.
Dependency direction is outward only: these utilities depend on `../utils/*`, `./qdrant/index.js`, `./memory/memory-accessors.js`, and `../tools/kairos-uri.js`; nothing in this module imports sibling files here.
