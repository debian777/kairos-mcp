Each subdirectory is an independent service with its own entry point; consumers import from `src/services/<service>/index.ts` (or the named file) rather than reaching into internals. Cross-service wiring happens at the application bootstrap: the metrics registry (`metrics/registry.ts`) is instantiated once and injected into every service that emits counters/histograms; the Redis-backed key-value store (`kv_and_shared_stores/key-value-store-factory.ts`) is shared between stats, OIDC state, proof-of-work, and execution-trace stores; the Qdrant client (`qdrant/service.ts`) is constructed once and passed down to both the memory store and the reward-propagation pipeline. Domain stores (`oidc-state-store`, `proof-of-work-store`, `forward-runtime-store`, `execution-trace-store`, `export-download-capability-store`) all compose on top of the same KV abstraction so they can be swapped between in-memory and Redis without changing their callers. The embedding provider, memory store, and stats scorer are the only services that reach out to external systems (OpenAI/TEI, Qdrant, Redis), while the remaining modules are pure or HTTP-only helpers used by multiple adapters.
