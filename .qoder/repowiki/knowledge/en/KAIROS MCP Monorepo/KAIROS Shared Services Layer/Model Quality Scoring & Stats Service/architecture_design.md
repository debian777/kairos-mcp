Single orchestrator class `ModelStatsService` in `model-stats.ts` owns in-memory state (`StatsState`, bonus totals, rare-success counts) and persists it via the shared `keyValueStore` (Redis-backed). It delegates pure computation to sibling modules with a clear dependency direction: `model-stats.ts` → `scoring.ts` (keyword-driven specificity/expert/broad/longevity scoring), `bonuses.ts` (Wilson-adjusted success-rate + rarity/comparative bonus), `healer.ts` (experience-scaled healer improvement bonus), and `protocol.ts` (lazy-imported aggregation of step `quality_metadata` from Qdrant). Types live in `types.ts`; `protocol.ts` is the only file that reaches outside this package (imports `qdrantService` lazily to avoid circular deps). The service exposes both pure-score helpers (`calculateQualityScore`, `calculateStepQualityMetadata`) and mutating flows (`processContribution`, `calculateImplementationBonus`, `calculateHealerBonus`) that also emit Prometheus metrics via `../metrics/agent-metrics.js` and record recent discoveries capped at 20 entries.
