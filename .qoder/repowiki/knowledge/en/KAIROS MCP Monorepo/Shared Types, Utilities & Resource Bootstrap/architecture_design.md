The scope is a flat collection of leaf modules with no internal sub-packages; each file is an independent module re-exported or imported by higher layers.

- `src/types/` — pure type declarations: `index.ts` defines the domain `KairosError`, `QualityMetrics`, implementation/healer stats, and protocol step success-rate shapes; `memory.ts` holds the in-memory store shape; `archiver-zip.d.ts` augments third-party typings. These are consumed by services/tools but do not import runtime code.
- `src/utils/` — runtime helpers grouped by concern:
  - Logging: `log-core.ts` creates a single shared Pino instance (redaction paths, text vs JSON formatter, stdio/stdout selection) via `getBaseLogger()`, while `structured-logger.ts` wraps it behind a `StructuredLoggerApi` with audit-line sanitization and an Express `httpLogger` middleware.
  - Qdrant vector DB: `qdrant-utils.ts` re-exports from `qdrant-vector-types.js`, `qdrant-collection-utils.js`, `qdrant-vector-management.js`, `qdrant-query-utils.js` as a single barrel for storage-layer calls.
  - Tenant/space: `tenant-context.ts`, `resolve-space-param.ts`, `protected-space-write-guard.ts`, `space-display.ts`, `space-filter.ts` provide multi-tenant scoping utilities.
  - Misc: `frontmatter.ts`, `version-compare.ts`, `protocol-slug.ts`, `uri-builder.ts`, `dump-markdown.ts`, `concurrency-limit.ts`, `global-error-handlers.ts`, `build-version.ts`, `normalize-redis-url.ts`, `zod-to-jsonschema.ts`, `mcp-tool-doc-runtime.ts`, `audit-log-events.ts`, `audit-mcp-summary.ts`, `kairos-local-artifact-dirs.ts`, `kairos-user-dirs.ts`, `memory-body.ts`, `memory-store-utils.ts`, `qdrant-*` helpers.
- `src/resources/` — bootstrap layer that wires up MCP resources at startup: `resource-bootstrap.ts` registers a throwaway resource/template so the SDK exposes `resources/*` even when none are present; `mem-resources-boot.ts` scans `embed-docs/mem/*.md`, compares shipped vs stored adapter versions, deletes stale entries, trains them into Qdrant, and persists a `content_sha256` payload; `mem-dir-utils.ts` resolves primary/fallback mem directories; `mem-uuid-mapper.ts`, `embedded-mcp-resources.ts`, `prompt-resources.ts`, `docs-resources.ts` supply concrete resource loaders.
- `src/constants/` — immutable constants such as `builtin-search-meta.ts` which derives footer labels from embedded meta docs rather than hardcoding strings.
- `src/config/` — environment-backed settings (`export-download-settings.ts`, `export-zip-settings.ts`, `adapter-markdown-size-limits.ts`) using small `envString`/`envInt` helpers with fallbacks.

Dependency direction is one-way: config → utils → resources → (consumed by) services/tools/controllers. The logging subsystem is the only cross-cutting dependency that all other modules may call back into.
